# Product

## Register

product

## Users

Hybrid traders and crypto analysts who query market data through natural language.
They need both density (speed to scan multiple signals) and depth (ability to drill
into a specific coin, regime, or signal). Context ranges from quick glance at a
dashboard to focused research on a single asset.

## Product Purpose

Saarthi translates natural language questions into precise crypto market intelligence.
It exists to bridge the gap between a user's intent ("show me top gainers") and
structured analytics (screener tables, ML signals, market regimes, news, coin
details, composite reports) without requiring SQL or dashboard configuration.

## Brand Personality

Hyperliquid meets Linear. Bold, clean, premium — crypto-native without the gaudy
overload. Confident without arrogance. The interface should feel like an elite
intelligence analyst explaining markets, not a tool dumping raw query results.

## Anti-references

- **Shadcn default look** — generic Tailwind UI with no distinct identity.
  Avoid any surface that could be mistaken for a starter template.
- Anything cluttered, ad-heavy, or overloaded with indicators.

## Design Principles

1. **Data-first clarity** — Every pixel serves understanding. Decoration must earn
   its place by making data clearer, not by filling space.

2. **Agent confidence** — The interface should feel like an intelligent assistant
   explaining markets, not a database query tool dumping results. Responses have
   conviction: visual hierarchy, explanatory narrative, clear state.

3. **Progressive disclosure** — Show the headline number first, reveal detail on
   demand. Never overwhelm at first glance. Each response type has a clear
   hero/pivot/explore flow.

4. **Consistent intelligence** — All 6 response types (screener, ML signals,
   market regime, news, coin detail, composite) should feel like they come from
   the same expert mind, not 6 different dashboards bolted together.

## Accessibility & Inclusion

Color-blind safe. Green/red signals must be reinforced with shape, pattern, or
text labels since crypto direction cues (up/down) rely heavily on color.
Standard WCAG AA for contrast and keyboard navigation.
